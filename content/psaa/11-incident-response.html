11 · Incident Response (IR)

IR Phases — PICERL

PhaseKey Actions
1. PreparationPolicies, tools, playbooks ready before incident
2. IdentificationDetect and confirm — is this real?
3. ContainmentStop the spread — isolate affected systems
4. EradicationRemove malware, close vulnerabilities
5. RecoveryRestore systems to normal operation
6. Lessons LearnedDocument what happened, prevent recurrence

Quick Response by Scenario

ScenarioResponse Steps
Malware on endpointIsolate → capture memory → hash malware → check persistence → eradicate
Suspicious network trafficCapture pcap → identify source/dest → check process → block → escalate
Phishing emailDon't click → analyze headers → check URLs/attachments → block sender → alert
Brute forceIdentify source IP → check success → block IP → reset affected accounts
Data exfiltrationCapture traffic → identify destination → block → preserve evidence → notify

Containment Strategies

StrategyWhen to UseTrade-off
Network isolationActive C2 communicationAttacker loses visibility — may trigger anti-analysis
Account disableCompromised credentialsDisrupts legitimate user
Firewall blockKnown bad IP/domainAttacker may pivot to new infrastructure
Process killActive malware runningLose memory evidence — capture RAM first!
Host isolationRansomware spreadingService disruption

IR Report — Key Sections

  1. Executive Summary — what happened, business impact, 1 paragraph
  2. Timeline — chronological sequence of events
  3. Technical Details — IOCs, TTPs, affected systems
  4. Root Cause — how did the attacker get in?
  5. Containment Actions — what was done to stop the spread
  6. Eradication Steps — what was removed/patched
  7. Recommendations — how to prevent recurrence
Exam Tip: The 24-hour PSAA exam requires a written report — structure it with these sections.
