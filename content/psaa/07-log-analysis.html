Core Log Analysis Commands

# Count total lines in log
wc -l logfile.log

# View last N lines
tail -20 logfile.log

# View first N lines
head -20 logfile.log

# Get last/latest entry (timestamp)
tail -1 logfile.log

# Follow log in real time
tail -f logfile.log

# Search for pattern
grep 'pattern' logfile.log

# Case insensitive search
grep -i 'pattern' logfile.log

# Search with line numbers
grep -n 'pattern' logfile.log

# Search multiple patterns (OR)
grep -i 'error\|fail\|denied' logfile.log

# Extract column 1 (IP addresses)
awk '{print $1}' logfile.log

# Extract quote-delimited field
awk -F'"' '{print $6}' logfile.log

# Count + sort unique values
sort | uniq -c | sort -rn

# Top 10 IPs from Apache log
awk '{print $1}' access.log | sort | uniq -c | sort -rn | head -10

# Count unique user agents
awk -F'"' '{print $6}' access.log | sort -u | wc -l

# List all user agents with counts
awk -F'"' '{print $6}' access.log | sort | uniq -c | sort -rn

# Find 404 errors with line numbers
grep -n ' 404 ' access.log

# Find SQL injection attempts
grep -i 'union\|select\|%27\|1=1\|--' access.log

# Count HTTP status codes
awk '{print $9}' access.log | sort | uniq -c | sort -rn

URL Encoding Cheat Sheet

Attackers encode to bypass filters — these appear in log files:

EncodedDecodedWhat it Means
%27'Single quote — starts SQL injection
%20(space)Space
%2D%2D--SQL comment — ends/terminates query
%3D=Equals sign
%3C<Less than — XSS opening tag
%3E>Greater than — XSS closing tag
%2F/Forward slash
%00NULLNull byte — path traversal
