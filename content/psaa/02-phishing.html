Phishing Analysis Methodology

  1. Initial Triage — quickly assess and prioritize severity
  2. Header & Sender Examination — investigate MTA addresses, IPs, true origin
  3. Context Examination — analyze language, formatting, social engineering red flags
  4. Web & URL Examination — collect artifacts, use tools to inspect URLs/domains
  5. Attachment Examination — securely extract and analyze attachments

Email Authentication — DKIM, SPF, DMARC

MethodWhat It DoesRed Flag

DMARCPolicy telling receivers what to do on SPF/DKIM fail"none" = not enforced

Linux Phishing Analysis Commands

# Hash attachment for VirusTotal lookup
sha256sum file.iso && sha1sum file.iso && md5sum file.iso

# Decode base64 encoded content
echo 'BASE64STRING' | base64 -d

# Extract URLs from email file
grep -oP 'https?://[^"]+' email.eml | sort -u

# View raw email headers
cat email.eml | head -60

# Check file type regardless of extension
file suspicious_attachment
