Order of Volatility — Collect Most Volatile First

Rule: Always collect from most volatile to least volatile. Rebooting destroys RAM!
OrderSourceWhy
1CPU registers/cacheLost immediately on shutdown
2RAM/MemoryLost on reboot
3Network connectionsChange constantly
4Running processesChange over time
5Disk/filesystemPersistent but can be overwritten
6LogsMay be rotated
7Backups/ArchivesMost stable

Forensics Commands

# Hash file for evidence integrity
sha256sum file

# View file metadata
stat filename

# Check file type
file suspicious_binary

# Check last logins
last

# Check failed logins
sudo lastb

# Find recently modified files
find / -mmin -60 -type f 2>/dev/null

# Find .bak files anywhere on system
find / -type f -name '*.bak' 2>/dev/null

# Check active connections
sudo ss -anp

# View bash history
cat ~/.bash_history

# View journal logs (systemd systems)
journalctl

# View journal for specific service
journalctl -u apache2

# View journal since last boot
journalctl -b

Chain of Custody Principles

  • Document everything — who touched evidence, when, why
  • Hash before and after — prove evidence wasn't tampered
  • Work on copies — never modify original evidence
  • Write-protect — use hardware blockers when imaging drives
    • TimelineReconstruct events in chronological order from multiple sources
